# @vibeflow/build-metadata

## What This Package Does

This package provides the build metadata (version, commit SHA, commit date, release channel) to the rest of the application. The metadata is injected at build time by `scripts/inject-build-metadata.ts` and exported from a generated file.

## What It Exports

- `BUILD_METADATA` — a typed constant with `version`, `commitSha`, `commitDate`, `releaseChannel`
- `getBuildMetadata()` — a function that returns the build metadata (same as the constant, but useful for testing)

## Who Depends On It

- `apps/desktop` main process (top bar display)
- `apps/desktop` renderer (top bar display)
- `electron-updater` integration (uses `version` for update checks)

## Dependencies

- None (pure TypeScript, no external dependencies)

## Generated File

The file `src/generated.ts` is auto-generated by `scripts/inject-build-metadata.ts` at build time.
It is listed in `.gitignore` and must NOT be committed to git.
It is regenerated fresh on every build and every `pnpm dev` start.

## Notes

- If `src/generated.ts` does not exist (e.g., fresh clone before first build), the package exports safe fallback values: `version: '0.0.0-dev'`, `commitSha: 'unknown'`, etc.
- See `/docs/build-metadata.md` for the full build metadata injection process
